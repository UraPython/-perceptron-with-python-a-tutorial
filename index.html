O Perceptron¶

Inventado em 1957 por Frank Rosenblatt no Cornell Aeronautical Laboratory, o Perceptron um tipo de rede neural de classificao simples, em outras palavras, um algoritmo que torna possvel separar elementos de grupos que tenham caractersticas diferentes e com base nessa separao prever a qual grupo pertencer elementos no classificados previamente. Como essa rede uma das mais simples, geralmente ela usada como porta de entrada do mundo do Machine Learning.

Como funciona¶

Imagem retirada do livro "Python Machine Learning" do Sebastian Raschka

O Perceptron funciona da seguinte forma: ele recebe entradas (usualmente chamados de inputs) de uma fonte externa, geralmente obtidas por meio de observao, realiza a soma do produto desses dados por pesos, que so aprendidos e otimizados. Os pesos so representados por w e os inputs so representados porx

Em seguida, ele passa esse valor encontrado para a funo de ativao, que, no caso dele, uma funo binria que testa o valor gerando resultados distintos para cada condio, sendo as condies: maior que o parmetro de comparao (1) e menor que o parmetro (-1). Esse parmetro de comparao geralmente chamado de limiar (em ingls, threshold). Por fim, o Perceptron libera como sada (usualmente chamado de output) a previso.

Para aprender esses pesos, o Perceptron passa por um processo de treinamento.

Como é feito o treinamento¶

Para o treinamento do Perceptron estabelecido uma taxa de aprendizado, este nmero representa o quanto o erro vai interferir no aprendizado, e uma taxa de iteraes, ou seja, as repeties que sero feitas. A cada iterao, usualmente chamada de poca, o erro das previses calculado por meio da frmula:

erro das previses = taxa de aprendizado * (valor esperado - previso)

Os erros calculados anteriormente so utilizados para atualizar o peso das entradas. Suponha que eu tenho um input x e um erro r, para cada x somado xr, ou seja, x + xr. O primeiro input geralmente o bias (em portugus, vis).

O Perceptron usa a aprendizagem supervisionada, mas alm dela existem outros tipos.

Tipos de Machine Learning¶

Existem 3 tipos de Machine Learning:

Aprendizagem supervisionada: nela os dados de treinamento possuem rtulos que representam suas caractersticas.

Aprendizagem no supervisionada: diferente da supervisionada, os dados no possuem rtulos.

Aprendizagem por Reforo: essa forma de aprendizado muito usada para tomada de decises e geralmente atribui-se recompensas e penalidades ao modelo de acordo com a ao que se quer priorizar.

O primeiro e mais importante o mtodo que servir para treinar a classe, o fit, que recebe como parmetro o conjunto de dados, chamado de dataset, com os inputs no parmetro X e os rtulos do dataset no y:

class Perceptron:
    def __init__(self, eta=0.01, n_iter=10):
        self.eta = eta
        self.n_iter = 10
        
    def fit(self, X, y):
        self.w_ = np.zeros(1 + X.shape[1])
        self.errors_ = []
        
        for _ in range(self.n_iter):
            errors = 0
            for xi, target in zip(X, y): 
                update = self.eta * (target - self.predict(xi))
                self.w_[1:] += update * xi
                self.w_[0] += update
                errors += int(update != 0)
            self.errors_.append(errors)

Aqui primeiramente iniciamos os pesos como um vetor de zeros e criamos a lista de erros. Depois abrimos o lao que vai repetir o treinamento para todas as pocas. Dentro dele iniciamos uma varivel errors ao valor 0 e percorremos o dataset usando o mtodo zip para juntar os inputs ao rtulo, a cada iterao pegamos os inputs na varivel xi e o rtulo na varivel target.

Em um segundo lao calculamos o update, que a funo de erro do Perceptron, e atualizamos os pesos com o update multiplicado pelo input. Note que a primeira posio do vetor de pesos o bias, por isso o update nela no multiplicado por nenhuma entrada. Por ltimo somamos o erro calculado ao total e fora do lao os adicionamos lista de erros, para salvar os erros dessa poca.

O mtodo fit chama o mtodo predict, falaremos dele agora. Ele recebe como parmetro um vetor de inputs:

Por ltimo temos o mtodo net_input, que recebe como parmetro um vetor de entradas:

In [6]:
class Perceptron:
    def __init__(self, eta=0.01, n_iter=10):
        self.eta = eta
        self.n_iter = 10
        
    def fit(self, X, y):
        self.w_ = np.zeros(1 + X.shape[1])
        self.errors_ = []
        
        for _ in range(self.n_iter):
            errors = 0
            for xi, target in zip(X, y): 
                update = self.eta * (target - self.predict(xi))
                self.w_[1:] += update * xi
                self.w_[0] += update
                errors += int(update != 0)
            self.errors_.append(errors)
        
    def predict(self, X):
        return np.where(self.net_input(X) >= 0.0, 1, -1)
    
    def net_input(self, X):
        return np.dot(X, self.w_[1:]) + self.w_[0]

Esse mtodo retorna o produto escalar entre o vetor de entradas xi e o vetor de pesos somado com o bias.

def plot_decision_region(X, y, classifier, resolution=0.02):
    markers = ('s', 'x', 'o', '^', 'v')
    colors = ('red', 'blue', 'lightgreen', 'gray', 'cyan')
    cmap = ListedColormap(colors[:len(np.unique(y))])
    
    x1_min, x1_max = X[:, 0].min() - 1, X[:, 0].max() + 1
    x2_min, x2_max = X[:, 1].min() - 1, X[:, 1].max() + 1
    
    xx1, xx2 = np.meshgrid(np.arange(x1_min, x1_max, resolution), np.arange(x2_min, x2_max, resolution))
    
    Z = classifier.predict(np.array([xx1.ravel(), xx2.ravel()]).T)
    Z = Z.reshape(xx1.shape)
    plt.contourf(xx1, xx2, Z, alpha=0.4, cmap=cmap)
    
    for idx, cl in enumerate(np.unique(y)):
        plt.scatter(x=X[y == cl, 0], y=X[y==cl, 1], alpha=0.8, c=cmap(idx), marker=markers[idx], label=cl)

Essa funo recebe um vetor de inputs, um vetor de rtulos, o classificador (nesse caso o Perceptron), e uma resoluo. Depois ela cria uma lista de cores e estilos do scatter do matplotlib e usa o meshgrid do NumPy para gerar uma malha com os pontos do intervalo entre os menores e os maiores valores dos inputs, com a resoluo passada por parmetro. Ento a varivel Z criada utilizando previses do Perceptron com os valores da malha e usa-se a funo contourf para colorir os pontos. Aps isso, um lao for criado para gerar o plot com os valores de y.
